Git使用详解（1）

  • 帮助文档

之前使用的版本控制工具是SVN，在朋友圈看到有个小老弟搭建了一个GitHub Pages博客，有点手痒，于是就开始研究。据了解，从事企业团队项目开发，都要涉及到版本控制系统。目前主流版本管理工具有SVN与Git。相较于SVN,Git的主要优势有：分布式部署，每个用户都可以成为版本库、无需联网，本地可提交代码、具备完善的分支体系、高性能等优点。目前大部分企业都沿用Git,因此对于Git的学习也是必然。

Git与GitHub

Git的使用

1. 创建版本库，进入需要创建的项目文件中鼠标右键右击菜单“Git Bash Here”命令；输入“git init”项目文件生成 .git目录
2. 提交文件到版本库，
(1)若在项目文件夹中新增一个名为index.php的文件,更新到版本库的命令为:git add index.php  
(2)也可使用git commit 命令，把添加的文件提交到版本库中

Unordered Lists() git commit -m “首页入口文件”
*配置系统的用户名、邮箱：
git config –global user.name “zhangyang”
git config –global user.email “228452914@qq.com“

3. Git查看版本库信息
(1)git status 及时查看项目文件的变化情况
(2)git diff 查看本次修改内容
(3)git commit -m '增加了一行代码' 对代码修改的描述
4. 日志查看与版本回退
（1）git log 查看最新提交日志
（2）git reset --hard commit-id 回退到指定版本号 （版本号建议在六位以上）
     git reset --hard HEAD~3 将文件最近三次提交回退

GitHub的使用

1. 官网：https://github.com/
（1）注册账号
（2）创建ssh key:进入C:\Users\Administrator目录鼠标右击打开Git Bash

Unordered Lists() 执行命令：ssh-keygen -t rsa -C “GitHub的注册邮箱”
id_rsa为私钥 id_rsa.pub为公钥（可对外使用）
（3）在GitHub上设置ssh密钥
Unordered Lists()
单击右上角头像选择Setting菜单项，再选择SSH and GPG keys，单击进入密钥设置界面，单击New SSH Key按钮，进入表单 Title可自定义
*Key为id_rsa.pub文件的密钥字符串

2. 与远程库关联
git remote add origin https://github.com/ZhangYang-smart/phpshop.git
3. 后续推送操作简化命令：
git push -u origin master
4. 复制远程库到本地
git clone git@github.com:ZhangYang-smart/phpshop.git

分支、合并与冲突解决

1. 新建一个dev分支
   git branch dev
2. 查看分支列表
   git branch (*标的一行，说明此分支正在使用)
3. 切换到dev分支
   git checkout dev
4. 在dev工作区下新建一个脚本文件 function.php
5. 执行新增和提交命令
   git add function.php
   git commit -m '自定义函数文件'
6. 切换到master分支
   git checkout master
  （发现dev分支下所开发的function.php脚本文件已消失不见）
7. 执行命令，合并dev分支到master,切换到master分支 
   git merge dev
8. git clean -d -fx表示:删除一些没有git add的文件
