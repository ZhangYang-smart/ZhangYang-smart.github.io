初涉 GitHub Pages

一时兴起，利用Hexo框架搭建了自己的个人博客，今后会使用git不定时的更新博客的内容。这是我的第一个博客，希望可以利用该博客分享自己的工作经验、个人感悟和随笔。希望得到大家的反馈，相互交流，共同学习！

搭建概要

使用Hexo框架生成静态网站

1.Node.js的安装：百度下载Node.js 64bit .msi
2.安装淘宝npm镜像 npm install -g cnpm --registry=https://registry.npm.taobao.org 
3.安装Hexo框架 cnpm install -g hexo
4.使用Hexo初始化博客项目 首先进入项目部署文件夹，执行命令
  hexo init myblog 
  npm install
5.初始化成功后，进入myblog目录下执行命令：
  hexo s
  可直接访问本地地址：localhost:4000

推送文章网站到GitHub Pages

1. 新建一个仓库（Create a new repository）
  仓库名为：用户名.github.io
2. 成功后返回myblog文件夹，执行命令：
  npm install hexo --save
  hexo generate 
  cnpm install hexo-developer-git --save
3. 完成后文本编辑器打开_config.yml文件，更改文件末尾配置项
deploy:
  type: git
  repository: git@github.com:用户名/用户名.github.io.git
  branch: master（注：' : '后面都要加一个空格，否则会报错）
4. 再执行命令：
  npm install --save hexo-deployer-git
  hexo  deploy
部署完成：https://zhangyang-smart.github.io/

为个人博客的配置与主题设置

1. _config.xml配置文件
 title：网站的标题
 subtitle：网站的副标题
 description：网站的描述
 author：网站的作者
 language：网站使用的语言
 timezone：网站的时区
2.主题设置：
 访问https://hexo.io/themes/
 例如设置主题为 yelee,执行命令：git clone https://github.com/MOxFIVE/hexo-theme-yelee.git themes/yelee
 再修改_config.xml配置文件，更换默认主题为yelee

Git使用详解（1）

之前使用的版本控制工具是SVN，在朋友圈看到有个小老弟搭建了一个GitHub Pages博客，有点手痒，于是就开始研究。据了解，从事企业团队项目开发，都要涉及到版本控制系统。目前主流版本管理工具有SVN与Git。相较于SVN,Git的主要优势有：分布式部署，每个用户都可以成为版本库、无需联网，本地可提交代码、具备完善的分支体系、高性能等优点。目前大部分企业都沿用Git,因此对于Git的学习也是必然。

Git与GitHub

Git的使用

1. 创建版本库，进入需要创建的项目文件中鼠标右键右击菜单“Git Bash Here”命令；输入“git init”项目文件生成 .git目录
2. 提交文件到版本库，
(1)若在项目文件夹中新增一个名为index.php的文件,更新到版本库的命令为:git add index.php  
(2)也可使用git commit 命令，把添加的文件提交到版本库中 
   git commit -m "首页入口文件" 
   配置系统的用户名、邮箱：
   git config --global user.name "zhangyang"
   git config --global user.email "228452914@qq.com"
3. Git查看版本库信息
(1)git status 及时查看项目文件的变化情况
(2)git diff 查看本次修改内容
(3)git commit -m '增加了一行代码' 对代码修改的描述
4. 日志查看与版本回退
（1）git log 查看最新提交日志
（2）git reset --hard commit-id 回退到指定版本号 （版本号建议在六位以上）
     git reset --hard HEAD~3 将文件最近三次提交回退

GitHub的使用

1. 官网：https://github.com/
（1）注册账号
（2）创建ssh key:进入C:\Users\Administrator目录鼠标右击打开Git Bash 
     执行命令：ssh-keygen -t  rsa -C  "GitHub的注册邮箱"
     id_rsa为私钥 id_rsa.pub为公钥（可对外使用）
（3）在GitHub上设置ssh密钥 
     单击右上角头像选择Setting菜单项，再选择SSH and GPG keys，单击进入密钥设置界面，单击New SSH Key按钮，进入表单 
     Title可自定义
     Key为id_rsa.pub文件的密钥字符串
2. 与远程库关联
git remote add origin https://github.com/ZhangYang-smart/phpshop.git
3. 后续推送操作简化命令：
git push -u origin master
4. 复制远程库到本地
git clone git@github.com:ZhangYang-smart/phpshop.git

分支、合并与冲突解决

1. 新建一个dev分支
   git branch dev
2. 查看分支列表
   git branch (*标的一行，说明此分支正在使用)
3. 切换到dev分支
   git checkout dev
4. 在dev工作区下新建一个脚本文件 function.php
5. 执行新增和提交命令
   git add function.php
   git commit -m '自定义函数文件'
6. 切换到master分支
   git checkout master
  （发现dev分支下所开发的function.php脚本文件已消失不见）
7. 执行命令，合并dev分支到master,切换到master分支 
   git merge dev
8. git clean -d -fx表示:删除一些没有git add的文件
